POSTGRESQL - INSTALLATION PARAMETRIX LAUNCH WITH A DIFFERENT VERSION FROM SETUP VERSION

    Problema: non si riesce a installare correttamente postgresql 9.6, versione del setup di Parametrix.

    Sintomi:

    • installando postgresql compare il seguente errore: failed to load sql modules

    • il servizio postgres non si avvia: potrebbe sembrare in esecuzione; facendo avvia/riavvia compare un messaggio che dice che si è avviato e poi è stato arrestato

    • Parametrix non si avvia, nel log si trovano errori riguardanti il database

    Articolari correlati:

    • https://stackoverflow.com/questions/67265537/postgresql-downgrade-password-encryption-from-scram-to-md5

    Soluzione:

    1. Disinstallare postgres

    2. Creare un utente locale “postgres” psw: postgres con privilegi di amministratore

    3. Installare una diversa versione di postegres (partire da postgresql-9.6.24-1 e andare a scendere, se non si riesce con queste provare con postgresql-13.5-1)

    4. Se non ci sono messaggi di errore procedere, altrimenti cambiare versione di postgres

    5. Nella cartella di installazione di postgres aprire il file "postgresql.conf":

      1. modificare "password_encryption" in "md5"

      2. salvare il file

    6. Nella cartella di installazione di postgres aprire il file "pg_hba.conf"

      1. modificare le prime 3 occorrenze di "scram-sha-256" in "trust"

      2. salvare il file

    7. riavviare il servizio

    8. Lanciare cmd.exe e digitare runas /user:postgres cmd.exe

    9. cd \

    10. \password postgres

    11. Inserire la password scelta

    12. Confermare la password scelta

    13. Nella cartella di installazione di postgres aprire il file "pg_hba.conf"

      1. modificare le prime 3 occorrenze di "scram-sha-256" in "md5"

      2. salvare il file

    14. riavviare il servizio

    15. cd \bin

    16. psql -c "CREATE USER donatoni WITH PASSWORD 'Parametr1x2017';"

    17. psql -c "ALTER USER donatoni WITH SUPERUSER";

    18. Trovare la chiave di registro: "Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\postgresql-x64-13." (il nome della chiave dipende dalla versione di posgresql installata; NB: la chiave deve esistere, altrimenti c'è stato un problema):

      1. modificare Display Name in: postgresql-9.6

      2. modificare Image Path: "\bin\pg_ctl.exe" runservice -N "postgresql-9.6" -D "\data" -w (dovrebbe cambiare solo la parte in grassetto)

      3. Controllare che esista la chiave Computer\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\PostgreSQL\Installations\postgresql-9.6:

        1. se non c'è crearla

        2. creare campo di tipo "Valore stringa": nome "Base Directory", valore ""

        3. riavviare il servizio

    ;
